# ABC-анализ продаж (SQL)

## 📌 Задача
Провести ABC-анализ товаров/услуг на основе данных о продажах.  
Выделить категории:
- **A** — товары, приносящие 80% выручки
- **B** — следующие 10% выручки
- **C** — оставшиеся 10% выручки

## 🛠 Инструменты
- SQL (с поддержкой оконных функций: PostgreSQL, SQLite 3.25+, MS SQL Server)
- Оконные функции: `SUM() OVER()`, `SUM() OVER(ORDER BY ...)`

## 📊 Данные
- Таблица: `SALESanal`
- Колонки: `Наименование`, `Количество`, `Сумма`

## 🔍 Логика запроса
1. Для каждой строки считается накопленная сумма продаж при сортировке от наибольшей суммы к наименьшей
2. Накопленная сумма делится на общую сумму всех продаж → получается накопленная доля
3. На основе доли присваивается категория:
   - ≤ 0.8 → **A**
   - ≤ 0.9 → **B**
   - > 0.9 → **C**

## 📈 SQL-запрос
```sql
SELECT 
    Наименование,
    Количество,
    Сумма,
    SUM(Сумма) OVER (ORDER BY Сумма DESC) * 1.0 / SUM(Сумма) OVER () AS накопленная_доля,
    CASE 
        WHEN SUM(Сумма) OVER (ORDER BY Сумма DESC) * 1.0 / SUM(Сумма) OVER () <= 0.8 THEN 'A'
        WHEN SUM(Сумма) OVER (ORDER BY Сумма DESC) * 1.0 / SUM(Сумма) OVER () <= 0.9 THEN 'B'
        ELSE 'C'
    END AS Группа
FROM SALESanal
ORDER BY Сумма DESC;
